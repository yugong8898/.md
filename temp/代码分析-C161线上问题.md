## 一、核心结论速览

1. **三个模块均无公共金额汇总组件**：各自在 `watch(selection)` 中用 `BigNumber` 直接累加接口返回字段，前端**未做任何加减修正**（即 `应付 = sum(字段)`，`运费差价 = sum(serviceCost)`）。
2. **确认存在 Bug 位置**：三处「运费差价」汇总均直接累加 `serviceCharge`（未减 `oilDiscountAmount`），与需求规则1/规则4（运费差价 = C - D）不符。
3. **重大风险提示（重复扣减）**：司机对账 `driverChecking/index.vue:1023` 有注释 `// 应付金额（后端已包含油气优惠的扣除，无需前端再次计算）`。若后端字段 `waybillToConsignorPrice` / `payableFpFreight` 实际已含油气扣除，前端**再套用 `应付 = A + E - D` 会重复扣减**。这恰恰是需求原文语义模糊、需后端确认字段口径之处（见不确定项）。
4. **字段名已基本定位**（候选值见第六节），但"未支付货主油气优惠"是否为独立新增字段、与现有 `oilDiscountAmount` 的关系仍需后端确认。
