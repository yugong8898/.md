# 【需求分析文档】C161 线上问题

## 1. 需求概述

修复网络货运单在【司机对账】、【运单付款-运单付款】、【运单付款勾选汇总】等场景下，应付/应支付/应到付/应回单付等金额未正确减去货主油气优惠、未正确包含货物保障服务金额的计算逻辑错误。

---

## 2. 涉及端和页面

- PC端：
  - 【司机对账】列表页（含勾选网货运单后的汇总区）
  - 【运单付款 - 运单付款】列表页（含【应支付】汇总值）
  - 【运单付款 - 运单付款】列表勾选后的【应到付】【应回单付】
  - 【运单结算处理】列表页（含勾选网货运单后的【应付】【运费差价】）
- 移动端/小程序：未提及，暂不涉及
- 公共组件：待排查是否有复用的金额汇总组件

---

## 3. 核心用户故事

- 作为【财务/结算人员】，我希望【司机对账列表中运费差价数值能正确减去货主油气优惠】，以便【应付金额与司机运费+运费差价保持一致、准确反映实际应付】
- 作为【财务/结算人员】，我希望【运单付款列表中应支付金额能减去未申请的货主油气优惠】，以便【准确统计待支付合计】
- 作为【财务/结算人员】，我希望【勾选网货运单后应到付、应回单付能正确计算（含货物保障服务费、减去油气优惠）】，以便【付款操作前金额准确】
- 作为【财务/结算人员】，我希望【运单结算处理列表应付和运费差价已含货主油气优惠和货物保障服务的正确口径】，以便【结算处理无偏差】

---

## 4. 业务规则和约束

变量定义（来自需求原文）：
- A = 应付运费（含税）
- B = 司机运费
- C = 运费差价
- D = 货主油气优惠
- E = 货主购买保费（货物保障服务金额）

**规则1 - 司机对账列表（勾选网货运单后汇总）：**
- 应付 = A + E - D
- 司机运费 = B（不变）
- 运费差价 = C - D（当前 Bug：未减去 D）

**规则2 - 运单付款-运单付款列表【应支付】：**
- 应支付 = 所有查询结果中【支付状态=待支付】的运单的（未申请支付司机金额 + 未申请运费差价 + 运单上未申请的货主沟通货物保障服务金额 - 运单上未支付的货主油气优惠金额）的合计
- 显示条件（已实现，不需要改）：当前登录企业未启用组织额度 且 查询条件选择了某个网货主体

**规则3 - 运单付款-运单付款列表勾选网货运单后：**
- 应到付 = A + E - D - 已申请预付金额（含税）+ 未申请货主保费 - 未申请货主油气优惠
- 应回单付 = 回单付金额中未支付金额（是否减油气优惠待确认）

**规则4 - 运单结算处理列表（勾选网货运单后汇总）：**
- 应付 = A + E - D
- 运费差价 = C - D（需确认是否已实现）

## 5. 边界情况和异常场景

- 场景1：运单无油气优惠（D=0）→ 预期行为：应付/运费差价计算结果与原逻辑一致，不受影响
- 场景2：运单无货物保障服务（E=0）→ 预期行为：应付计算正常，不加 E
- 场景3：油气优惠 D 大于运费差价 C（C-D < 0）→ 预期行为：待确认（展示负数 or 显示 0）
- 场景4：勾选混合运单（含网货/非网货）→ 预期行为：油气优惠和保障服务是否只对网货运单生效待确认
- 场景5：应支付汇总中部分运单油气优惠未申请、部分已申请 → 预期行为：仅减去"未支付/未申请"部分，不重复扣减

## 6. 字段和接口说明（已知部分）

- `货主油气优惠`（D）：字段名待确认，含义为货主享受的油气补贴优惠，来源：运单明细/列表接口
- `货物保障服务金额`（E）：字段名待确认，含义为货主购买的货物保障险费用，来源：运单明细/列表接口
- `未申请运费差价`：字段名待确认，来源：运单列表接口
- `未申请支付司机金额`：字段名待确认，来源：运单列表接口
- `未支付货主油气优惠`：字段名待确认（C161 新增字段），来源：运单列表接口
- 相关接口路径：待排查，涉及司机对账、运单付款、运单结算处理三个模块

## 8. 验收标准

- ✅ 场景1：司机对账列表，勾选含油气优惠的网货运单 → 运费差价列值 = C - D，应付 = A + E - D
- ✅ 场景2：运单付款列表应支付汇总 → 值 = 所有待支付运单的（未申请司机金额 + 未申请差价 + 未申请保障服务 - 未支付油气优惠）合计
- ✅ 场景3：运单付款列表，勾选网货运单 → 应到付 = A + E - D - 已申请预付 + 未申请保费 - 未申请油气优惠
- ✅ 场景4：油气优惠 = 0 时，各模块金额计算与原逻辑一致，无差异
- ✅ 场景5：运单结算处理列表，确认应付和运费差价已按正确公式计算（含 E、减 D）
