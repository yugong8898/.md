## 二、列表页（index.vue）

### 2.1 搜索区字段

搜索区支持展开/收起，通过 `isExpand` 控制，`getAllFieldRows()` 按 4 列分组，前 3 行默认展示。

| 字段名 | 参数 | 类型 | 说明 |
|--------|------|------|------|
| 派车日期 | `sendCarDate` / `sendCarDateEnd` | 日期范围 | 时间范围限制 ±2 个月，不能选未来日期；watch 拼接时分秒 |
| 创建日期 | `unloadTimeStartDate` / `unloadTimeEndDate` | 日期范围 | 同上 |
| 运单号 | `batchBusBillId` | 文本 | 最多 200 个，空格隔开；格式校验：只能含字母/数字/空格；**有值时派车/创建时间不生效** |
| 车牌号/船名航次/铁路运单号 | `busNumber` | 文本+下拉 | 右侧 select 切换单个/批量（`multipleBusNumber`: 0/1）；单个模式自动 trim |
| 司机/承运商 | `dcCarrierCompanyName` | 文本 | trim |
| 状态 | `busStatusList` | 多选下拉 | 数据源：`driverCheckingEnumeration.statusMultiple` |
| 结算主体 | `payeeNames` | 文本 | trim |
| 运输类型 | `transportType` | 单选下拉 | 数据源：`driverCheckingEnumeration.driverChecking.transportType` |
| 出发地省市区/目的地省市区 | `sendAddrShorthand` | 文本 | labelWidth=120；trim |
| 出发地简称 | `originAbbrLike` | 文本 | trim |
| 目的地简称 | `destAbbrLike` | 文本 | trim |
| 订单号 | `orderId` | 文本 | trim |
| 货运类型 | `freightType` | 单选下拉 | 数据源：`driverCheckingEnumeration.typeOfFreight` |
| 网络货运主体 | `networkMainBodyName` | 自定义下拉 | 前端右模糊过滤 `networkMainBodyFilter()`；`visible-change` 时重置过滤列表 |
| 货源别名 | `orderAlias` | 文本 | trim |
| 运单别名 | `waybillAlias` | 文本 | trim |

**运单号优先逻辑**（在 `getTable` 内 `$nextTick` 执行）：

if (batchId) {

## 三、列表表格

使用 `table-drag` 组件，`uuid='DRIVER_CHECKING'`，列显示状态持久化到 localStorage。共 **58 列**，以下列出关键列。

### 3.1 关键列说明

| prop | 说明 | 特殊处理 |
|------|------|---------|
| `checkbox` | 全选/单选 | 全流程互斥校验；`handleSelectable()` 控制禁用；`isFullProcessConflict()` 控制 tooltip |
| `waybillId` | 运单号 | 点击跳转运单详情（`waybillIdClick`）；`expiryLimitStatus=2` 显示黄色图标，`=3` 显示红色图标 |
| `sendCarDate` | 派车日期 | `formatToDate().format('yyyy-MM-dd HH:mm')` |
| `receiptDate` | 回单日期 | `formatToDate().format('yyyy-MM-dd')` |
| `preExamTime` | 预审通过时间 | `formatToDate().format('yyyy-MM-dd HH:mm')` |
| `$startEnd` | 出发地-目的地 | `$startEnd(row)` 拼接 `sendAddrShorthand + '-' + receiveAddrShorthand` |
| `$all` | 应付金额 | `$all(row)` 格式化 |
| `transportCost` / `otherCost` 等金额列 | 多个金额列 | `parsePrice()` 格式化；`serviceCharge`/`waybillToConsignorPrice` 当 `companyType===1` 时显示 `--`，否则 `v-mask-star` 脱敏展示 |
| `unitRice` | 单价 | `parsePrice(unitRice) + '(' + exGetUnitPerFixName(settlementType) + ')'` |
| `settlementQuantity` | 结算数量 | `exGetgoodsQuantity() + exGetUnitName()` |
| `gx` | 柜型柜数 | 仅 `transportType=1/2` 时显示；`containerNumber + '*' + codeFormat('gx', containerType)` |
| `splitCollectionInfo` | 委托转账主体 | tooltip 显示名称/账号（可脱敏切换）/支付渠道 |
| `splitType` | 委托转账规则 | `splitFlag='10'` 时显示；`renewTimes>0` 标红，`splitType='10'` 黑色，否则橙色 |
| `platformCode` | 来源 | `sourceObj[platformCode] \|\| platformCodeName \|\| '--'` |
| `fullProcessFlag` | 是否全流程 | `=10` 显示"是"，否则"否" |
| `shipperCompanyName` | 上游客户名称 | `companyType===2` 时显示，否则 `--` |
| `payableB2Freight` | 应付B2运费 | `companyType===1` 时显示 `payableFpFreight`，否则 `--` |
| `b1PayableFreight` | 客户应付运费 | `companyType===2` 时显示 `payableFpFreight`，否则 `--` |
| `taxRate` | 税率 | `isNumber(taxRate)` 时显示 `taxRate + '%'`，否则 `--` |
| `expenseDetailNote` / `goodsRemark` | 备注类字段 | 超 20 字截断显示 `...`；tooltip 每 50 字换行展示完整内容 |
| `expiryLimitStatus` | 时效状态 | `expiryLimitStatusObj` 映射 |
| `handle` | 操作列 | 见下方操作按钮规则 |

### 3.2 操作列按钮

操作列按钮均通过 `$route.meta.xxx`（权限）和 `$action(row, 'xxx')`（状态）双重守卫控制显示。

| 按钮 | 权限字段 | `$action` 条件 | 方法 |
|------|---------|--------------|------|
| 查看 | `viewDetail` | `busStatus` 任意 | `viewDetail('read', id, waybillId)` |
| 核对 | `verify` | `busStatus='100'`（待核对） | `viewDetail('verify', id, waybillId)` |
| 审核 | `check` | `busStatus='110'`（待审核） | `viewDetail('check', id, waybillId)` |
| 退回 | `giveBack` | `busStatus='130'`（审核通过） | `giveBackConfirm(row)` → 退回弹窗 |
| 推送 | `push` | `busStatus='130'` | `push(row)` → 推送弹窗 |
| 对账确认 | `affirm` | `busStatus='200'`（待对账） | `affirmConfirm(row)` → 对账确认弹窗 |

### 3.3 顶部操作栏

| 元素 | 条件 | 说明 |
|------|------|------|
| 走马灯公告 | `showCarouse=true` | 内容来自 `getParameterByParamName` 接口；文字长度决定容器宽度（small/normal/large） |
| 费用说明按钮 | 始终显示 | 打开费用计算规则弹窗（`FEE_DETAIL` 静态数据） |
| 导出 | `$route.meta.export` | `ExportBtn` 组件；`exportBefore()` 处理列映射和脱敏 |
| 批量确认 | `$route.meta.batchAffirm` | `batchAffirm()` |
| 批量不通过 | `$route.meta.batchDisPass` | `handleBatchDisPass()` |

### 3.4 底部选中统计

```
选中 N 单
装货数量：按 settlementType 分组累加 loadQuantity
卸货数量：按 settlementType 分组累加 unloadQuantity
应付：全流程 → 累加 payableFpFreight；非全流程 → 累加 waybillToConsignorPrice
含司机运费：累加 receivableTotalCost
运费差价：累加 serviceCharge（全流程 B1 不参与）
```

### 3.5 全选/单选逻辑

**单选（`el-checkbox-group` v-model `selectedIds`）**
- `selectedIds` 变化 → watch 反查 `reseponse.model` → 同步更新 `selection[]`
- `handleSelectable(row)` 返回 false 时禁用：已选中运单的 `fullProcessFlag` 与当前行不一致

**全选（`handleCheckAllChange`）**
- 检查当前页所有行的 `fullProcessFlag` 是否一致，不一致则阻止并提示
- 全选时将所有行的 `waybillId` 写入 `selectedIds`

**冲突 tooltip（`isFullProcessConflict`）**
- 当前行未被选中 且 与已选行 `fullProcessFlag` 不一致时返回 true，显示提示文案

## 四、业务逻辑

### 4.1 状态机（busStatus）

```
100  待核对   → 操作：核对(verify)
110  待审核   → 操作：审核(check) / 批量不通过
115  处理中   → 操作：查看(read) only
120  审核不通过 → 操作：查看 only
130  审核通过  → 操作：查看 / 退回(giveBack) / 推送(push)
200  待对账   → 操作：查看 / 退回 / 对账确认(affirm)
210  对账退回  → 操作：查看 only
220  对账通过  → 操作：查看 only
```

状态 → 可用操作由 `index.format.js` 中的 `$action(row, action)` 控制。

### 4.2 页面初始化流程（activated）

```
1. 从 $route.meta 读取按钮权限 → controlBtn
2. loadStatusOption()        加载枚举字典 + 来源下拉
3. fetchOrganizationList()   加载所属组织，写入 copyCarrierOrgIdList
4. 等待 orgTree.fetchPromise 判断是否有组织权限
5. getHandlerList()          加载经办人列表
6. getAndApplyWorkspaceParams()  从工作台缓存恢复筛选条件
7. getTable()                查询列表（所属组织无权限则跳过）
8. getNotice()               获取走马灯公告
```

### 4.3 列表查询（getTable）

```
searchForm.validate()
  → getUseNew('query_waybill_account')   判断是否走 ES 接口
  → apiMethod(params)                    getTable 或 _getTable
  → 返回数据补充 showFlagSplit=false
  → 重置 selection / selectedIds
```

**ES 开关逻辑（getUseNew）**：
1. 先查平台参数「司机对账es开关」，值为 `'10'` 则全量走 ES
2. 否则查字典 `query_waybill_account` 白名单，当前 `companyId` 在列表内则走 ES

### 4.4 批量确认（batchAffirm）

```
校验：selection.length > 0
筛选：busStatus ∈ ['110','130','200']  有效运单 validList

全部不符合 → $confirm 提示"全部不符合条件"
部分符合   → $confirm 提示"X条符合，是否操作符合的？"
全部符合   → openBatchAffirmDialog(validList)

openBatchAffirmDialog：
  按 fullProcessFlag 分两套口径计算汇总金额
    fullProcessFlag=10：累加 receivableTotalCost / fpFreightDiffAmt / payableFpFreight
    其他：累加 receivableTotalCost / serviceCharge / premiumAmount / oilDiscountAmount / waybillToConsignorPrice
  打开弹窗 batchDialog=true

batchAffirmDialog（确认）：
  调 constManagement.driverChecking.batchAffirm({ waybillAccountsIdList, busStatus:'220' })
```

### 4.5 批量不通过（handleBatchDisPass）

```
校验：selection.length > 0
筛选：busStatus = '110'（仅待审核）validList

全部不符合 → 提示
部分符合   → $confirm → 打开弹窗
全部符合   → 直接打开弹窗

batchDisPassFn（确认）：
  调 constManagement.driverChecking.batchReviewNotPass(params)
  params = validList.map(i => ({
    bmsReview: { reviewResult:'20', reviewRemark: reason },
    versionCode, waybillAccountsId, waybillId
  }))
```

### 4.6 单笔退回（giveBack）

```
giveBackConfirm(row)  → show.dialog=true
giveBack()
  → detailFormEl.validate()
  → constManagement.driverChecking.giveBack({ waybillAccountsId, busStatus:'300', reason, waybillId })
```

### 4.7 单笔推送（push / pushOnline）

```
push(row) → 打开推送弹窗 pushShow.dialog=true

pushOnline()
  isPlatform=10 && settlement=1  → 线上推送
    constManagement.driverChecking.push({ waybillId, settlementId, waybillAccountsId })
  其他组合                       → 线下推送（生成文件）
    constManagement.driverChecking.pushForFile({ waybillId, waybillAccountsId, busStatus, downloadFlag:2 })
```

### 4.8 单笔对账确认（affirm）

```
affirmConfirm(row)
  网络货运(freightType='2') → 展示汇总金额（showtotalAffirm=true）
    普通：serviceCharge / premiumAmount / waybillToConsignorPrice
    全流程：fpFreightDiffAmt / payableFpFreight
  打开弹窗 affirmShow.dialog=true

affirm()
  → 组装 bmsAttachmentList（来自 frontPhoto 上传结果）
  → constManagement.driverChecking.affirm({ waybillAccountsId, busStatus:'220', bmsAttachmentList, bmsReview, settlementName })
```

### 4.9 导出（exportBefore）

```
1. 读取 localStorage[uuid] 自定义列配置
2. filter checked=true，去掉操作列和序号
3. 虚拟 prop 映射：
   '$startEnd' → 'sendAddrShorthand'
   '$SettlementModes' → 'settlementMethod'
   'gx' → 'containerName'
   '$all' → 'amountsPayable'
4. insertParams(propArr, 'unitRice', 'settlementType')  插入结算类型
5. showStar=true 时替换脱敏字段名：
   'waybillToConsignorPrice' → 'waybillToConsignorPriceEncryption'
   'serviceCharge'           → 'serviceChargeEncryption'
6. bizType = '3pl-bmsWaybillAccountList-new'
```

## 六、详情页（add.vue）

路由：`/userBase/driverChecking/:action/:id`，`action` 取值 `read` / `verify` / `check`，三种模式复用同一组件。

### 6.1 三种模式对比

| 维度 | read（查看） | verify（核对） | check（审核） |
|------|------------|--------------|-------------|
| 费用表确认金额 | 只读展示 | **可编辑输入** | 只读展示 |
| 费用表备注 | 只读 | **可编辑** | 只读 |
| 审核结果表单 | 不显示 | 不显示 | **显示且可填** |
| 保存按钮 | 隐藏 | 显示（需权限 `save`） | 隐藏 |
| 提交审核按钮 | 隐藏 | 显示（需权限 `commit`） | 隐藏 |
| 确定按钮 | 隐藏 | 隐藏 | 显示（需权限 `confirm`） |
| 返回按钮 | 显示 | 显示 | 显示 |

### 6.2 页面初始化（mounted）

```
loadStatusOption()         加载枚举字典（运输类型、开户行、费用类型等）
getPartPay()               获取分次付款数据（预付/回单付/到付）
getDetail()
  → constManagement.driverChecking.getDetail(request)
  → 按 transportType + settlementType 动态替换 formList[9~11]
    （结算重量/结算数量/单价的 label 和 prop 随结算类型变化）
  → 处理 bmsBusinessCostList：补空小计行、格式化金额（busCost / updBusCost）
  → fetchOilGasConfig()    获取油气返利配置（决定油气优惠行是否显示）
  → $total()               计算并追加特殊汇总行
  → getRecord()            加载操作记录（仅 createDate >= 2024-02-01 的运单）
```

### 6.3 运单信息区（formList）

布局固定 4 列一行，默认 6 行（24 个字段）；无委托转账时（`splitFlag !== '10'`）删除 2 项改为 5 行。

字段 `prop` 按 `type` 分两类：
- `type='text'`：直接读 `reseponse.model[prop]`，el-tooltip 展示完整内容
- `type='slot'`：按 `prop` 分支做格式化、拼接、点击交互

关键 slot 字段处理：

| prop | 展示逻辑 |
|------|---------|
| `waybillId` | 蓝色可点击，跳转运单详情 |
| `payeeNames` | 蓝色可点击，调 `getSettlementDetail()` |
| `sendCarDate` / `zcsj` / `receiptDate` | `formatToDate().format('yyyy-MM-dd HH:mm')` |
| `$startEnd` | `$startEnd(sendAddrShorthand, receiveAddrShorthand)` 拼接 |
| `busNumber` | 铁路/水运直接展示；公路显示蓝色跳转车辆监控；微应用内不跳转 |
| `goodsName` | `renderRowInfo(model)` 渲染货物明细 |
| `settlementQuantity` / `settlementWeight` | `exValidatorWeight()` 格式化 |
| `unitRice` | `parsePrice()` |
| `loadWeight` / `unloadWeight` | 蓝色可点击，调 `handleLookImg()` 查看附件照片 |
| `$otherKuiTonsRatio` | 亏涨吨情况，见下方说明 |
| `splitCollectionInfo` | 委托转账主体：名称/账号（可脱敏切换）/支付渠道 |
| `splitType` | 委托转账规则：全额运费 或 自定义金额+单位；颜色同列表页规则 |

**亏涨吨情况（`$otherKuiTonsRatio`）四种表现**：

| 表现 | 出现条件 |
|------|---------|
| 未设置亏吨免赔 | `otherKuiTonsRatioShow=true` 但 `lsdsGoodsDeductible.enable !== 1` |
| 空白 | 免赔已开启，但 `loadWeight`/`unloadWeight` 缺值、`transportType=1/2` 或 `kuiTonsSituationVO.lossType` 无值 |
| 亏吨/涨吨 XX 千克 | 免赔已开启，`kuiTonsSituationVO` 有有效的 `lossType`/`lossNumber` |
| 正常 | 同上，`lossType` 为"正常" |

### 6.4 费用信息区（bmsBusinessCostList）

使用 `primitive-table` 组件，列配置来自 `add.conf.js`。

**普通费用行**（非特殊行）：

| 列 | verify 模式 | read/check 模式 |
|----|-----------|----------------|
| 序号 | 从 1 开始 | 同左 |
| 费用项目 | 只读 | 只读 |
| 费用类型 | `codeFormat('costType', val)` | 同左 |
| 原金额 | `parsePrice(parsePriceReduction(busCost))` | 同左 |
| 确认金额 | **`el-input` 可编辑**，`@input` 触发 `updBusCostInput` | 只读文本 |
| 费用备注 | **`el-input` 可编辑** | 只读 |
| 附件 | `$files` 展示附件列表 | 同左 |

**特殊汇总行**（由 `add.format.js $total()` 追加，仅 `action !== 'verify' && freightType === '2'` 时）：

通过 `specialRowType` 字段标识，首列合并"序号+费用项目"两列（`spanMethod` 控制）：

| specialRowType | 显示文字 | 确认金额取值 | 显示条件 |
|----------------|---------|------------|---------|
| `subtotal` | 小计 | 前端累加计算 | 始终 |
| `freightDiff` | 运费差价 | `serviceCharge`（v-mask-star 脱敏） | `companyType !== 1` |
| `insurance` | 货物保障服务金额 | `premiumAmount`（全流程为 0） | `premiumBuyerType === '10'` |
| `oilDiscount` | 油气优惠 | `-oilDiscountAmount`（-1 时显示 --） | 有有效油气数据 或 全流程（金额为0） |

| `carrierPay` | 应付承运商运费 | `payableFpFreight`（加粗） | 全流程 `companyType === 1` |

特殊行备注列的额外内容：

- `oilDiscount`：显示"油气实际使用 X 元"
- `fpDiff`：显示"差价率：X%"
- `shipperPay`：显示"上游客户名称：XXX"

### 6.7 核对流程（verify 模式）

```
保存（save）
  → form.validate()
  → constManagement.driverChecking.save(request)

提交审核（verify）
  → form.validate()
  → constManagement.driverChecking.verify(request)
  → 成功后 goBack()
```

### 6.8 审核流程（check 模式）

```
确定（check）
  → form.validate()
  → constManagement.driverChecking.check(request)
    request 包含 bmsReview: { reviewResult, reviewRemark }
  → 成功后 goBack()
```
