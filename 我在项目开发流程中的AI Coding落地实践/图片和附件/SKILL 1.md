# 浏览器自动化UI测试

## 工作流程

### Step 1：确定测试目标

根据用户输入判断测试来源：

- **用户给了路由地址** → 直接跳到 Step 3 打开页面
- **用户描述了功能点** → 生成测试用例后执行
- **用户无具体描述** → 分析 `git diff` 当前变更，自动生成测试用例

### Step 2：生成测试用例

若需要自动生成，执行以下分析：

```bash
git diff --name-only
git diff --stat
```

根据变更文件和代码内容，生成结构化测试用例：

```
## 测试用例清单

### TC-01: [功能名称]
- 前置条件：[页面状态/数据要求]
- 操作步骤：
  1. [具体操作]
  2. [具体操作]
- 预期结果：[期望行为]
- 优先级：P0/P1/P2
```

**用例生成规则：**

| 变更类型          | 测试重点                        |
| ----------------- | ------------------------------- |
| 新增搜索/筛选字段 | 字段交互、联动、清空、重置      |
| 新增表单          | 必填校验、提交、回显            |
| 新增列表列        | 数据渲染、排序、空值处理        |
| 新增按钮/操作     | 点击行为、权限控制、loading状态 |
| 修改已有功能      | 回归原功能 + 验证新逻辑         |
| 修复Bug           | 复现路径 + 修复后验证           |

### Step 3：构建页面URL

#### 微前端项目路由规则

本项目是 qiankun 微前端子应用，**必须**通过主应用访问：

| 子应用标识 | 主应用地址              | URL拼接规则                                 |
| ---------- | ----------------------- | ------------------------------------------- |
| `app-base` | `http://localhost:5678` | `http://localhost:5678/app-base/{路由路径}` |
| `app-core` | `http://localhost:5678` | `http://localhost:5678/app-core/{路由路径}` |

#### URL构建逻辑

1. 检测当前项目：查看项目根目录名或 `package.json` 的 `name` 字段
2. 如果是 `app-base` 或 `app-core`：
   - 基础URL = `http://localhost:5678/{子应用标识}/{用户给的路由路径}`
3. 如果用户给了完整URL，直接使用
4. 如果用户只给了路由路径（如 `synchronizeWaybill/list`）：
   - 拼接为 `http://localhost:5678/app-base/synchronizeWaybill/list`

#### 从代码推断路由

如果用户没给路由地址，从以下途径推断：

- 查看 `src/router/routes.ts` 中的静态路由
- 查看变更文件所在目录路径（如 `src/views/synchronizeWaybill/waterway-waybill/` → 路由可能是 `synchronizeWaybill/waterway-waybill`）
- 搜索 `path:` 关键字在路由配置中的定义

### Step 4：启动浏览器执行测试

使用 Browser Agent 执行测试，构建 prompt 时**必须**包含以下内容：

#### 4.1 自动登录处理

在 Browser Agent prompt 中加入登录检测逻辑：

```
如果页面是登录页（包含用户名/密码输入框）：
1. 先尝试账号：sysadmin，密码：cs123456
2. 如果登录失败，再尝试账号：super，密码：cs123456
3. 登录后重新导航到目标页面
```

#### 4.2 Browser Agent Prompt 模板

```
## 任务：UI功能自动化测试

### 导航
1. 打开 {目标URL}
2. 等待页面加载完成（最多10秒）
3. 截图查看当前状态

### 登录检测
如果看到登录页面（用户名/密码输入框）：
- 输入用户名：sysadmin
- 输入密码：cs123456
- 点击登录按钮
- 等待登录完成并跳转到目标页面
- 如果 sysadmin 登录失败，用 super / cs123456 重试

### 测试用例执行
{逐个列出测试用例，每个用例包含：}
- 具体操作步骤（点击、输入、选择等）
- 每步操作后截图
- 记录实际结果 vs 预期结果

### 输出要求
返回：
1. 每个测试用例的执行结果（通过/失败）
2. 每步操作的截图路径
3. 发现的UI问题描述
```

### Step 5：输出测试报告

汇总 Browser Agent 返回结果，以表格形式展示：

```
## 测试报告

| # | 用例 | 结果 | 截图 |
|---|------|------|------|
| 1 | [用例描述] | ✅/❌ | [截图] |

### 问题清单
| # | 严重程度 | 描述 | 截图 |
|---|---------|------|------|
| 1 | 高/中/低 | [问题描述] | [截图] |
```

## 注意事项

- **微前端项目必须通过主应用端口访问**，直接访问子应用端口可能无法正常加载路由
- 浏览器操作间需留足等待时间（页面渲染、接口响应）
- 每个关键操作步骤都要截图，便于问题追溯
- 如遇页面白屏或加载异常，先检查浏览器控制台是否有报错
- 登录凭证仅用于本地开发环境测试
