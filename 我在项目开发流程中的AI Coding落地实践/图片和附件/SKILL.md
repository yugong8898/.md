# 前端技术方案设计文档生成

通过交互式问答收集信息，按模板生成完整的前端技术方案设计文档。

## 交互流程

按以下阶段逐步引导用户，每阶段用 AskUserQuestion 或对话收集信息：

### 阶段1：基本信息

收集以下信息（可从附件或上下文推断）：

- **需求名称**：本次方案对应的需求/功能名
- **需求背景**：简要说明为什么要做这个需求
- **关联文档**：需求文档/任务拆解文档链接（如有）
- **涉及模块**：列出受影响的前端模块及核心文件路径

### 阶段2：详细设计

针对每个功能点收集：

- **实现方案**：组件层级、数据流向、关键逻辑
- **涉及文件变更**：文件路径 + 变更类型（新增/修改/删除）+ 变更说明
- **交互说明**：用户操作 → 前端行为 → 接口调用 → UI反馈

如果用户提供了任务拆解文档，按其中的模块/子任务逐一展开设计。

### 阶段3：接口与数据

- **接口列表**：方法、路径、请求/响应结构
- **状态管理**：是否涉及 Store 新增/变更
- **组件通信**：props/emit/provide-inject/store

### 阶段4：专项设计（按需）

根据需求特征，选择性收集：

- **校验策略**：不同场景的校验规则差异
- **权限控制**：权限码定义、实现方式
- **性能优化**：懒加载、缓存策略
- **兼容性**：对现有功能的影响分析

### 阶段5：补充信息

- **测试要点**：核心场景 + 预期结果
- **排期里程碑**：开发/联调/自测时间
- **待确认项**：需要与后端/产品确认的问题

## 输出规则

1. **先展示后保存**：生成文档后先在对话中展示给用户确认
2. **确认后保存**：用户确认满意后，调用 create-doc skill 将文档保存到 `D:\docs\{需求ID}\` 目录
3. **可迭代修改**：用户提出修改意见后调整内容，再次确认
4. **智能推断**：如果用户提供了任务拆解文档或上下文，尽可能从中推断信息，减少提问次数
5. **按需裁剪**：模板中的章节根据实际需求裁剪，不强制所有章节都填写

## 注意事项

- 章节编号根据实际保留的章节重新排序
- 代码示例使用 TypeScript + Vue3 Composition API 风格
- 文件路径使用项目内相对路径或绝对路径
- 权限码、接口路径等信息若未确认，标注 `[待确认]`
