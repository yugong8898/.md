### B1 补充说明

方案只提到 `confirmMoney` 写错行。实际 L557 `this.$set(this.newList[0], 'confirmRemark', rate)` 也是**无条件执行**的：

```js
// L554-557 实际代码
if (this.detailObj.companyType !== 1) {
  this.$set(this.newList[0], 'confirmMoney', this.serviceRateMoney)  // 有守卫
}
this.$set(this.newList[0], 'confirmRemark', rate)  // ← 无守卫，B1 时也执行
```

B1 企业（companyType=1）时 `newList[0]` 是"货物保障服务金额"行，差价率会错误写入该行的 confirmRemark。

### B4 补充说明

L1054 的 `dealZeroModeFn()` 在 if/else **外面**，意味着：
- 删除抹零行 → 弹 $confirm → 用户点"取消" → `.catch(() => {})` → **dealZeroModeFn 仍然执行**
- 删除抹零行 → 弹 $confirm → 用户点"确定" → `getZeroDelStatus(index)`（异步）→ **dealZeroModeFn 已经用旧数据算完了**

两种情况都有问题。

### B5 补充说明

实际校验逻辑是 `Map<costType, costCode[]>` + `Set` 判重：

```js
// L1109-1126 实际代码
const map = new Map()
this.params.model.forEach(res => {
  if (res.costType && res.costName) {
    if (!map.has(res.costType)) {
      map.set(res.costType, [res.costCode])
    } else {
      const old = map.get(res.costType)
      map.set(res.costType, [...old, res.costCode])
    }
    for (const val of map) {
      if (new Set(val[1]).size !== val[1].length) {
        this.$message({ type: 'error', message: '相同费用类型和费用名称的费用项只允许录入一行' })
        n = '1'
        return false  // ← 只退出 forEach 回调，不退出 commit()
      }
    }
  } else {
    this.$message({ type: 'error', message: '费用类型或费用项目未填写完整' })
    n = '2'
  }
})
// forEach 结束后，不管 n 是什么，form.validate 仍然执行
this.$refs.form.validate((boolean, object) => { ... })
```

最终 L1153 `if (!n && ...)` 会正确返回错误标记，**但 validate 白跑了一次**。

## 七、P1/P2 阶段补充注意

### P1-2 油气接口 debounce

- 参考 `doCalculateFullProcessFreight`（L529-544）的 debounce 写法：`方法名: debounce(async function() {...}, 300)`
- 竞态保护用 `this._oilGasRequestId` 递增计数

### P1-4 newList 构建统一

- mounted 中的 newList 初始化（L455-487）可移到 watcher 首次触发时统一处理
- 注意 mounted 还做了 `window.addEventListener('resize', ...)` 和路由参数读取，这些不能移

### P2-1 计算纯函数提取

- `formatService` 与 `fomatFloatRoundMode` 有细微差异：
  - `formatService`：`(BigNumber(num) * 10^n).toFixed(7)` → 字符串 → Math.ceil/floor/round → 除回
  - `fomatFloatRoundMode`：`BigNumber(num).multipliedBy(10^n).toNumber()` → Math.ceil/floor/round → 除回 → 补零到固定小数位
- 提取时需确认用哪个精度策略，建议以 CostMessageDetails 的 `formatService` 为准（它更简单）
- endOfCalculation.js **不能改**（13+ 文件引用）

### P2-4 父子通信改 emit

- L862、L1547 两处 `dealZeroModeFn()` 是"父→子"方向（父主动触发子组件计算），emit 方向不对
- 建议：保持 $refs 调用，但改名为公开 API `recalcZeroMode()`，在组件文档中标注为 public method
- `commit()` 同理保持 $refs，作为对外校验 API
