## 四、两处必须修正的方案代码（照搬会回归）

### 重点 1：P0-1 已被"半修复"，方案"现状"描述过时

- 方案把 L554-557 当作"完全未修复"的硬编码下标。
- 现状：L554 已经加了 `if (companyType !== 1)` 守卫，但 L557 `confirmRemark` 仍是 `newList[0]`。
- 修正写法（同时覆盖两边，删掉冗余守卫）：

```js
countMoney(cost, rate) {
  if (!rate) rate = 0
  const total = this.count(cost, rate) + this.detailObj.premiumAmount
  this.serviceRateMoney = new BigNumber(total - cost - this.detailObj.premiumAmount).toFixed(2)
  const freightDiff = this.newList.find(item => item.costName === '运费差价')
  if (freightDiff) {
    if (this.detailObj.companyType !== 1) {
      this.$set(freightDiff, 'confirmMoney', this.serviceRateMoney)
    }
    this.$set(freightDiff, 'confirmRemark', rate)
  }
  this.$emit('newServiceCharge', this.serviceRateMoney)
  this.doCalculateFullProcessFreight()
}
```

### 重点 2：P0-5 方案"改后"片段缺 `n` 赋值，会放行脏数据

- 方案"改后"用裸 `for...of` + `return false`，意图直接中断 `commit()`。但 `return false` 会让 `commit()` 返回布尔 `false`，而 `edit.vue` L1224 判断的是 `CostDetails === '1' || CostDetails === '2'`。
- 后果：`commit()` 返回 `false` → 不等于 `'1'`/`'2'` → `edit.vue` 走到 `changeMoney(false)` → `false.map` 直接崩溃，或带着未校验数据继续保存。
- **当前代码其实已经正确**：`n` 在 `forEach` 内被赋成 `'1'`/`'2'`，结尾 `return n` 符合契约。真正的瑕疵只是 `forEach` 内的 `return false` 是无效的控制流（不会跳出循环，但 `n` 已设置，行为正确）。
- 修正写法（最小改动，保持返回契约）：把 `forEach` 内的 `return false` 换成"置 `n` + `break` 内层循环 + `return` 跳出当前 forEach 回调"，并确保最终 `return n`：

```js
commit() {
  let n = ''
  const map = new Map()
  outer: for (const res of this.params.model) {
    if (res.costType && res.costName) {
      if (!map.has(res.costType)) map.set(res.costType, [res.costCode])
      else map.set(res.costType, [...map.get(res.costType), res.costCode])
      for (const val of map.values()) {
        if (new Set(val).size !== val.length) {
          this.$message({ type: 'error', message: '相同费用类型和费用名称的费用项只允许录入一行' })
          n = '1'
          break outer
        }
      }
    } else {
      this.$message({ type: 'error', message: '费用类型或费用项目未填写完整' })
      n = '2'
      break outer
    }
  }
  // ... 原有 validate / request 构建逻辑不变 ...
  if (!n && this.params.model.length !== 0) return request
  return n
}
```

> 关键：用 `break outer` 跳过 `for...of`，函数正常落到 `return n`，**绝不**直接 `return false`。`edit.vue` L1224 的判断无需改动即可继续生效。
