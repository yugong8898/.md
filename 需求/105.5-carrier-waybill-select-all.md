# 105.5 承运人运单付款 - 全选/取消全选功能优化

## 需求背景

原有全选按钮只能执行全选操作，无法取消全选。本次调整为 toggle 逻辑：

- 全部选中时，点击全选 → 取消全选
- 部分选中或未选中时，点击全选 → 全部选中
- 文案保持"全选"不变

## 实施过程中发现并修复的额外问题

### 问题一：部分选中时全选不生效（历史遗留 Bug）

**现象**：部分选中状态下点击全选，UI 没有任何变化

**原因**：原有判断逻辑使用 `Array.every()` 判断是否全选，空数组时 `every()` 返回 `true`，导致判断结果错误

**修复**：改用计数比较方式精确判断

```javascript
// 修复前
const allChecked = targetList.every(item => item.checked)

// 修复后
const checkedCount = targetIndexes.filter(index => this.contentList[index].checked).length
const allChecked = checkedCount === targetIndexes.length
```

---

### 问题二：手动操作后全选/取消全选 UI 不响应（架构设计缺陷）

**现象**：手动勾选或取消某条数据后，再点击全选/取消全选，该条数据 UI 不再变化，但底部汇总数据（选中 X 单）显示正确

**原因**：`manageItem` 子组件里 `checked` 是 data + watch 的双状态设计，手动操作后子组件先改自己的 `this.checked`，父组件再写回 `dataPool.checked` 时，如果新值与当前值相同，`watch` 不会触发，UI 不响应

**修复**：将 `checked` 从 `data + watch` 改为 `computed`，完全由 `dataPool.checked` 驱动，彻底消除双状态

```javascript
// 修复前（双状态，存在不一致风险）
data() {
  return { checked: false }
},
watch: {
  'dataPool.checked'(val) {
    this.checked = val  // 值相同时不触发
  }
}

// 修复后（单状态，始终与父组件同步）
computed: {
  checked() {
    return !!this.dataPool?.checked
  }
}
```

---

### 问题三：`handleSelectionChange` 替换对象导致响应式失效（连锁问题）

**现象**：`checked` 改为 computed 后，单条手动勾选 UI 也不响应

**原因**：`handleSelectionChange` 使用 `$set(this.contentList, index, {...})` 替换整个对象，导致 `manageItem` 的 `dataPool` prop 变成新对象，新对象属性不是响应式的，computed 无法追踪变化

**修复**：改为直接修改对象属性，保持原对象引用

```javascript
// 修复前（替换整个对象，破坏响应式引用）
this.$set(this.contentList, params.index, {
  ...this.contentList[params.index],
  [params.key]: params.data
})

// 修复后（修改属性，保持引用，响应式正常）
this.$set(this.contentList[params.index], params.key, params.data)
```

## 最终改动详情

### `manageItem.vue`

1. `checked` 从 `data` 改为 `computed`，完全由 `dataPool.checked` 决定
2. 删除 `watch: 'dataPool.checked'`
3. `checkboxChange` 删除 `this.checked = !this.checked`（computed 不可直接赋值）
4. `updateChecked` 发送取反值 `!this.checked` 给父组件

### `carrierWaybillList.vue`

1. `selectAll` 实现 toggle 逻辑（原始需求）
2. `selectAll` 修复全选判断逻辑（问题一）
3. `handleSelectionChange` 改为修改属性而非替换对象（问题三）
