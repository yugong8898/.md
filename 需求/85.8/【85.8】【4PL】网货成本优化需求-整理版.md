## 详细需求说明

### 一、客户成本差价率页面

**原型地址**: https://rp.mockplus.cn/rps/oIgMTU8TU/dw1TrOwWE

#### 1.1 查询条件优化

**核心变更**: 业务归属公司拆分为两个级联查询条件

- **业务归属一级公司**: 可选集团本部 + 所有一级公司
- **业务归属二级公司**: 可选全部、暂无、选中一级公司下的所有二级公司

#### 1.2 查询逻辑规则

##### 1.2.1 集团本部场景

**条件**: 业务归属一级公司选择"集团本部"

**规则**: 
- 业务归属二级公司只能选择"全部"
- 显示全部数据(不考虑其他条件)

##### 1.2.2 一级公司场景

**条件**: 业务归属一级公司选择某个一级公司

**规则**:

| 二级公司选择 | 数据范围 |
|------------|---------|
| 全部 | 显示该一级公司下所有二级公司的客户成本差价率数据 |
| 暂无 | 显示业务归属公司为该一级公司(无二级公司)的客户成本差价率数据 |
| 具体二级公司 | 显示业务归属公司为该二级公司的客户成本差价率数据 |

##### 1.2.3 多条件组合

加上其他查询条件时,与以上结果取**交集**

#### 1.3 权限控制规则

##### 1.3.1 用户配置为集团本部

- 业务归属一级公司: 可选集团本部、所有一级公司
- 业务归属二级公司: 根据一级公司选择动态变化

##### 1.3.2 用户配置为一个或多个一级公司

- 业务归属一级公司: 只能选择配置的一级公司
- 业务归属二级公司: 可选全部、暂无、选中一级公司下的所有二级公司

##### 1.3.3 用户配置为一个或多个二级公司

- 业务归属一级公司: 只能选择配置的二级公司所属的一级公司
- 业务归属二级公司: 只能选择选中一级公司对应的配置的二级公司

##### 1.3.4 用户配置为一级公司+二级公司混合

- 业务归属一级公司: 只能选择配置的一级公司 + 配置的二级公司对应的一级公司

**细分规则**:

| 一级公司选择 | 二级公司可选范围 |
|------------|----------------|
| 用户配置的一级公司 | 全部、暂无、该一级公司下的所有二级公司 |
| 用户配置的二级公司对应的一级公司 | 仅当前一级公司下用户配置的二级公司 |

#### 1.4 列表字段变更

**原字段**: 业务归属公司

**新字段**: 
- 业务归属一级公司
- 业务归属二级公司

**显示规则**:

| 客户业务归属公司类型 | 一级公司字段 | 二级公司字段 |
|-------------------|------------|------------|
| 一级公司 | 显示该业务归属公司 | 为空 |
| 二级公司 | 显示该公司的上一级公司 | 显示该业务归属公司 |

#### 1.5 导出功能(新增)

**功能**: 按当前查询条件导出全部数据

**权限**: 单独权限,默认配置给超级管理员 **TODO 权限 ？**

### 二、服务费结算页面

**原型地址**: https://rp.mockplus.cn/rps/oIgMTU8TU/ebfo5kF40

#### 2.1 按运单页面

##### 2.1.1 查询条件优化

**与客户成本差价率页面相同**,业务归属公司拆分为一级/二级公司,级联逻辑和权限控制规则完全一致

##### 2.1.2 列表字段变更

**新增字段**:
- 业务归属一级公司
- 业务归属二级公司
- **业务类型**(新增)

**业务类型枚举值**:
- 统一定价主体
- 默认业务
- 轻卡业务
- 短倒业务
- 轻卡业务&短倒业务

**显示规则**: 与客户成本差价率页面相同

##### 2.1.3 按一级公司生成对账单(核心功能)

**功能描述**: 支持按业务归属一级公司批量生成该一级公司下所有二级公司的对账单

**操作流程**:

1. 选择查询条件:
   - 网货主体
   - 业务归属一级公司(选择某个一级公司)
   - 业务归属二级公司(选择"全部")
   - 待对账运单/已开票运单等其他条件

2. 点击"生成对账单"按钮

3. 进入生成对账单页面

4. 点击"确定",系统按运单归属的最低一级业务归属公司拆分数据,生成对账单

**校验规则**:

| 校验场景 | 提示信息 | 是否进入生成页面 |
|---------|---------|----------------|
| 按当前条件无数据 | "按当前条件查询不到数据" | 否 |
| 存在冲突的创建中对账单 | "按当前条件存在冲突的对账单在创建中,请等待冲突的对账单生成后再试" | 否 |

**冲突条件定义**:
- 同网货主体
- 同一个归属公司
- 开票时间有交叉
- 对账单状态为创建中

**特殊规则**:

1. 按一级公司生成对账单时,该一级公司下任一二级公司的对账单生成条件与创建中的对账单冲突,整个任务都不可执行

2. 按某个一级公司生成对账单时,如果没有归属该一级公司下某个二级公司的运单,则不生成该二级公司的对账单

3. 选择"集团本部"时,不可生成对账单

#### 2.2 按对账单页面

##### 2.2.1 查询条件和列表字段

**与按运单页面相同**,业务归属公司拆分为一级/二级公司

##### 2.2.2 导出功能(新增)

**功能**: 按查询条件导出对账单明细

**导出格式**: 压缩包

**导出限制**: 最多100个对账单

**超限提示**: "导出失败,最多可导出100个对账单的明细,请缩小范围"

**压缩包内容**:
- 每个对账单的运单明细数据(单独文件)
- 文件名称: 对账单编号

## 性能要求

### 批量生成对账单性能指标

**场景**: 按业务归属一级公司创建各业务归属二级公司对账单

**数据量**: 全部数据条数不超10万

**性能要求**: 所有对账单创建时间之和不超1小时

**计算公式**: 
```
总耗时 = Σ(各对账单生效成功时间 - 创建时间) ≤ 1小时
```

**验证方式**: 上线后,由业务产品、业务老师在生产环境以实际创建时间完成验证

### 数据迁移要求

**影响范围**:

- 服务费结算历史数据

**处理方式**: 
历史数据中的"业务归属公司"字段需要拆解为"业务归属一级公司"和"业务归属二级公司"

**拆解规则**:

| 原业务归属公司类型 | 业务归属一级公司 | 业务归属二级公司 |
|------------------|----------------|----------------|
| 一级公司 | 原业务归属公司 | 空 |
| 二级公司 | 该公司的上一级公司 | 原业务归属公司 |

## 技术实现要点

### 1. 级联选择器实现

```javascript
// 伪代码示例
{
  一级公司选择变化时: {
    if (选择 === '集团本部') {
      二级公司选项 = ['全部']
      二级公司默认值 = '全部'
      二级公司禁用 = true
    } else {
      二级公司选项 = ['全部', '暂无', ...获取该一级公司下的所有二级公司]
      二级公司禁用 = false
    }
  }
}
```

### 2. 权限过滤逻辑

```javascript
// 伪代码示例
function 获取可选一级公司列表(用户配置) {
  if (用户配置 === '集团本部') {
    return ['集团本部', ...所有一级公司]
  } else if (用户配置包含一级公司) {
    return [...用户配置的一级公司, ...用户配置的二级公司对应的一级公司]
  } else {
    return [...用户配置的二级公司对应的一级公司]
  }
}
```

### 3. 批量生成对账单逻辑

```javascript
// 伪代码示例
async function 按一级公司生成对账单(一级公司ID, 其他条件) {
  // 1. 查询该一级公司下所有运单
  const 运单列表 = await 查询运单(一级公司ID, 其他条件)
  
  // 2. 按最低一级业务归属公司分组
  const 分组数据 = 运单列表.groupBy(运单 => 运单.最低一级业务归属公司)
  
  // 3. 并发创建各二级公司对账单
  const 创建任务 = []
  for (const [公司ID, 运单] of 分组数据) {
    创建任务.push(创建对账单(公司ID, 运单))
  }
  
  await Promise.all(创建任务)
}
```

### 4. 冲突检测逻辑

```javascript
// 伪代码示例
function 检测对账单冲突(网货主体, 归属公司列表, 开票时间范围) {
  for (const 公司 of 归属公司列表) {
    const 创建中对账单 = 查询创建中对账单({
      网货主体,
      归属公司: 公司,
      开票时间交叉: 开票时间范围
    })
    
    if (创建中对账单.length > 0) {
      return { 冲突: true, 冲突公司: 公司 }
    }
  }
  
  return { 冲突: false }
}
```

## 风险点

### 1. 性能风险

**风险**: 按一级公司批量生成对账单时,数据量过大导致超时

**应对措施**:
- 限制单次生成的数据量上限(10万)
- 异步处理,提供进度查询
- 数据库查询优化,添加必要索引

### 2. 数据一致性风险

**风险**: 历史数据拆分时,一级/二级公司关系错误

**应对措施**:
- 数据迁移前做好备份
- 编写数据校验脚本
- 灰度发布,先在测试环境验证

### 3. 并发冲突风险

**风险**: 多用户同时按同一一级公司生成对账单

**应对措施**:
- 增加分布式锁
- 冲突检测逻辑覆盖所有二级公司
- 提供友好的冲突提示

### 4. 权限控制风险

**风险**: 复杂的权限规则导致用户看不到应该看到的数据

**应对措施**:
- 详细的权限配置文档
- 权限配置界面增加说明
- 提供权限测试工具
