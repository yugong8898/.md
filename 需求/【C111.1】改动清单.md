# 【C111.1】账户流水优化改动清单

## 需求概述

针对两个账户流水页面（`3pl.vue` 和 `mainstay.vue`），新增四个筛选条件（运单号、司机姓名、司机手机号、车牌号）、列表新增对应字段并支持导出、以及在选择具体账户类型+类型+用途时展示当前筛选条件下的金额汇总统计。

| 入口 | 路由 | 文件 |
|---|---|---|
| 基础-钱包总览-账户流水 | `/userBase/mainstay/3pl/turnover` | `turnover/3pl.vue` |
| 基础-网货钱包-账户流水 | `/userBase/mainstay/userBase/mainstay/turnover` | `turnover/mainstay.vue` |

## 二、功能点说明

### 2.1 功能点 1：新增筛选条件

在现有筛选区域新增以下四个筛选项：

| 筛选项 | 接口字段名 | placeholder | 查询方式 | 限制 |
|---|---|---|---|---|
| 运单号 | `busBillId` | 请输入运单号 | 精准查询 | 无长度限制 |
| 司机姓名 | `driverName` | 请输入司机姓名 | 精准+模糊查询 | 最多20个字符，超出不可输入 |
| 司机手机号 | `driverPhone` | 请输入司机手机号 | 精准查询 | 最多11位数字，超出不可输入，只能输入数字 |
| 车牌号 | `carNumber` | 请输入车牌号 | 精准查询 | 最多8位字符；`el-select + remote` 远程搜索，颜色展示在下拉选项 label 中；**传参只传 `carNumber`，不传颜色** |

**车牌号交互说明（参考 `platformweb/src/views/infolook/customerAggregating/components/SearchForm.vue`）：**
- 使用 `el-select`（`filterable + remote`），用户输入车牌号触发远程搜索接口
- 下拉选项 label 格式：`粤A12345(黄色)`，颜色名称通过 `cpys`（车牌颜色字典 `'267'`）映射，`logisticsweb` 中已有该枚举定义
- 选中后从 value 中解析出 `carNo`（车牌号）和 `carColor`（颜色 code），分别作为独立字段传参
- 颜色不需要单独的下拉，嵌入在搜索结果 label 里展示，用户选中即确定颜色
- 车牌远程搜索接口：`logisticsweb` 侧是否有对应接口待确认（`platformweb` 使用的是 `getCarListByPlateNo`，`logisticsweb` 中未找到同名接口）

### 2.2 功能点 2：列表新增字段

在现有列表基础上新增以下四列：

| 字段 | 字段名（待确认） | 宽度 | 是否固定 |
|---|---|---|---|
| 运单号 | `waybillNo`（待确认） | 180 | `fixed: 'left'`，固定在列表左侧 |
| 司机姓名 | `driverName`（待确认） | 120 | 否 |
| 司机手机号 | `driverPhone`（待确认） | 140 | 否 |
| 车牌号 | `plateNo`（待确认） | 120 | 否 |

**导出说明：**
- 新增字段需支持导出
- 导出使用现有 `ExportBtn` 组件，传参结构不变：`{ bizType: '3pl-accountFlowMeterPageList', filterModel: params }`
- `filterModel` 绑定的是 `params` computed，新增筛选字段加入 `params` 后自动透传给导出服务（`/exp/exportData`），**前端不需要额外改动**
- 导出的 Excel 列字段由后端导出模板控制，需后端同步在 `3pl-accountFlowMeterPageList` 对应的导出模板中增加运单号、司机姓名、司机手机号、车牌号四列，并支持接收新筛选字段用于过滤

### 2.3 功能点 3：统计信息展示

**展示条件（满足全部才展示）：**
- 账户类型 = 现金账户（`accountType === '0'`）
- 类型 ≠ 全部（`businessType !== ''`）
- 用途 ≠ 全部（`businessTypeSubset !== ''`）

**不展示条件（满足任一则不展示）：**
- 账户类型选「全部」或「授信账户」
- 类型选「全部」
- 用途选「全部」
- 默认进入页面时

**展示格式：**
```
现金账户  充值  预充值  000,000,000,000.00
```
- 金额使用红色字体标注
- 展示区域用 `v-if="summaryInfo"` 控制，`summaryInfo` 为 `null` 时不渲染

**触发/清空时机：**

| 操作 | 统计状态 |
|---|---|
| 点击查询，满足展示条件 | 调用统计接口，展示结果 |
| 点击查询，不满足展示条件 | `summaryInfo = null`，区域消失 |
| 点击重置 | `summaryInfo = null` |
| 切换账户类型（`changeDicListAll`） | `summaryInfo = null` |
| 切换类型（`changeDicList`） | `summaryInfo = null` |

## 四、改动内容

### 4.1 `3pl.vue` — 筛选区域

**旧逻辑注释保留，不删除。**

新增筛选行（第三行），包含以下四项：

1. **运单号**：`el-input`，无长度限制，`v-model.trim="formInline.busBillId"`
2. **司机姓名**：`el-input`，`maxlength="20"`，`v-model.trim="formInline.driverName"`
3. **司机手机号**：`el-input`，`maxlength="11"`，通过 `@input` 过滤非数字字符，`v-model.trim="formInline.driverPhone"`
4. **车牌号**：`el-select`（`filterable + remote`），参考 `SearchForm.vue` 的 `queryPlate` 实现：
   - 用户输入触发远程接口搜索车辆列表
   - 选项 label 格式：`粤A12345(黄色)`，颜色名称通过 `enumeration.cpys`（字典 id `'267'`）映射
   - 选项 value 直接为 `carNo`（车牌号字符串）
   - 选中后写入 `formInline.carNumber`；清空时同步清空
   - `data` 中新增：`plateStr: ''`（select 显示值）、`plateOptions: []`、`plateLoading: false`、`carColorObj: {}`（颜色字典映射，仅用于展示 label，不传接口）
   - `created` 中通过 `this.$store.dispatch('enumeration/getDirctionaryIds', [enumeration.cpys])` 初始化 `carColorObj`

## 九、风险点

| 风险 | 说明 | 建议 |
|---|---|---|
| 运单号 fixed 与序号列冲突 | 序号列无 fixed 属性，运单号加 fixed:left 后横向滚动可能出现样式错位 | 测试时重点验证；必要时序号列也加 fixed:left |
| 导出与列表不一致 | 后端导出接口未同步支持新筛选字段时，导出结果不含新条件过滤 | 导出接口与列表接口同步上线 |
| 统计和列表请求时序 | 若统计为独立接口且并发请求，可能出现统计先展示、列表还在 loading 的视觉不一致 | 串行：列表接口成功后再请求统计 |
| 统计展示条件判断前移到服务端 | 接口文档确认统计触发由服务端判断并返回 `display` 字段，前端无需预判，直接根据 `display` 渲染，减少了前端判断遗漏的风险 | 前端只判断 `display === true`，不在调用前做三个条件的前置 if |
