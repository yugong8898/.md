# 【C147.9】调整结算信息支持修改回单付金额 — 改动清单

## 需求概述

本次需求分为三块：

1. **付款方式改造**：调整结算/查看结算/司机对账查看支持展示「预付款、到付、回单付」，其中调整结算页支持修改回单付金额。

## 二、付款方式通用业务规则

### 2.1 字段说明

| 字段 | 是否可编辑 | 规则 |
|---|---|---|
| 预付款 | 不可编辑 | 运单有预付款则展示具体金额，没有则展示 0 |
| 到付 | 不可编辑 | 自动计算：`到付 = 小计 - 预付款 - 回单付` |
| 回单付 | 调整页可编辑，查看页只读 | 范围 `0 ~ 999999999.99`，最多两位小数 |

### 2.2 初始值规则

进入调整结算页面时：

- 回单付如果已经调整过：展示最新调整值。
- 回单付如果未调整过：默认继承货源设置的回单付金额。
- 运单原本未设置回单付：也允许手动输入回单付金额。
- 预付款固定只读。
- 到付根据当前小计、预付款、回单付自动计算。

> ⚠️ 不确定：后端需提供字段区分「未调整过」和「已调整为 0」，否则无法准确判断是否要继承货源默认回单付。

### 2.3 小计来源规则

小计用于计算：

```text
小计 = 到付 + 预付款 + 回单付
到付 = 小计 - 预付款 - 回单付
```

注意：

- PC 端小计需要从费用明细的实时确认金额合计中获取。
- 移动端已有 `subtotalConfirm` computed，可作为实时小计使用。
- 全流程运单的小计口径需要后端/产品确认，避免取到「合计」或「全流程应付」等特殊行。

> ⚠️ 高风险不确定：PC 端 `CostMessageDetails.vue` 是否直接暴露 `subtotal` 需要编码前确认。若没有，需要通过事件回调或父组件自身计算得到实时小计。

### 2.4 联动规则（重点）

#### 场景 1：小计变化

触发来源：修改费用明细确认金额、单价、结算数量等导致小计变化。

处理规则：

- 自动重算到付：`到付 = 小计 - 预付款 - 回单付`
- 如果计算后到付 `>= 0`：更新到付。
- 如果计算后到付 `< 0`：**到付不更新**，展示错误提示。
- 此场景下**不清空回单付**。

错误提示：

```text
应付司机运费不等于到付+预付+回单付的值
```

#### 场景 2：回单付变化

触发来源：用户手动修改回单付。

处理规则：

- 自动重算到付：`到付 = 小计 - 预付款 - 回单付`
- 如果计算后到付 `>= 0`：更新到付。
- 如果计算后到付 `< 0`：**到付不更新**，回单付行/输入框标红，展示错误提示，**回单付清空为 0**。

错误提示：

```text
应付司机运费不等于到付+预付+回单付的值
```

> 已确认：需求中「同时清空回单付的值」按 **清空为 0** 处理。

### 2.5 保存/提交校验

点击保存和提交时，需要新增校验：

#### 校验 1：等式校验

```text
小计 = 到付 + 预付款 + 回单付
```

不满足时提示：

```text
应付司机运费不等于到付+预付+回单付的值，请修改
```

实现注意：

- 必须使用 `BigNumber` 计算，不能使用原生浮点加减。
- 比较时建议允许 0.01 内误差，避免金额精度问题导致误判。

#### 校验 2：回单付比例校验

```text
回单付比例 = 回单付 / 小计
```

如果大于 4PL 配置比例，提示：

```text
回单付最大值为：小计*比例
```

实现注意：

- 小计为 0 时需要做除零保护。
- 4PL 比例来源待后端确认。

### 2.6 金额计算要求

- PC 端使用项目现有 `BigNumber` / `this.BNumber()`。
- 移动端使用项目现有 `BigNumber` 工具。
- 回单付、到付统一保留两位小数。
- 输入限制：非负数、最多两位小数、不超过 `999999999.99`。

## 十一、风险点

| 风险 | 说明 | 规避方式 |
|---|---|---|
| 小计来源错误 | 会导致到付计算、保存校验错误 | 编码前确认 PC 小计来源；移动端使用 `subtotalConfirm` |
| 回单付初始化错误 | 无法区分未调整和已调整为 0 | 后端提供状态字段或明确规则 |
| 浮点精度问题 | 等式校验误判 | 使用 BigNumber |
| 全流程运单口径特殊 | 小计可能不是普通运单小计 | 单独验证全流程运单 |
| 旧数据源误用 | 司机对账/付款页可能还用 partPay 旧字段 | 后端明确最新字段，前端统一替换 |
| Tab 改造影响表单状态 | 移动端调整页切 Tab 后数据丢失 | 用 `v-show` 保留 DOM，避免销毁表单 |
| 磅单接口字段不确定 | 前端无法最终编码 | 等后端接口文档确认后再接入 |
