# 【C149.5】易达宝 PC 端调整结算信息页面支持快速定位、装卸照片前置透出改动清单

## 需求概述

本次需求针对 PC 端「运单结算处理」中的调整结算信息页面（edit.vue）与查看调整信息页面（read.vue）：

- 在页面顶部增加一排固定导航按钮，点击跳转对应模块锚点
- 将装卸货磅单图片前置展示在结算信息模块内，采用左图右表单双栏布局，支持收起/展开
- 本版本不涉及图片增删改，也不涉及保存行为变更
- 结算信息模块抽成公共组件 `SettlementInfo.vue`，edit.vue 与 read.vue 均接入
- > 🆕 **【范围变更】** read.vue（查看调整信息）由「本期不动」调整为**本期同步处理**，与 edit.vue 同步接入锚点导航 + SettlementInfo 组件 + 磅单图片前置

**结算信息模块整体布局：左图右表单**

结算信息模块采用左右双栏布局（`display: flex`）：

- 左侧：磅单图片区域，固定宽度
- 右侧：原有表单字段，`flex: 1` 占剩余宽度
- 右上角有"收起装卸照片"按钮，点击折叠左侧图片区域，表单自动回到全宽；再次点击展开

**磅单图片展开/收起规则：**

| 状态 | 说明 |
|---|---|
| 默认 | 始终展开，无论有无图片 |
| 无图片 | 左侧显示空状态文案，不自动收起 |
| 收起 | 用户手动点击右上角按钮，左侧隐藏，表单全宽；按钮文案变为"展开装卸照片" |
| 展开 | 用户手动点击再次展开；按钮文案变为"收起装卸照片" |

**磅单图片部分：**

- 接口：复用现有接口 `fetchWeighBillImages`（`/waybill-attachment-query/find-attachment-by-waybill-id-status`），装货/卸货分两次 `Promise.all` 并行请求；装货 `attachmentTypes: ['1']`，卸货 `attachmentTypes: ['2']`；`status` 按运输类型由内部常量 `WEIGH_BILL_STATUS_MAP` 决定（transportType 1/2 → 装 410/卸 430，transportType 3/6 → 装 410/卸 420），不在映射中的运输类型（如铁路/水运）直接重置为空图片；响应 `model[].attachmentUrl` 为 OSS path，经 `getReceiptUrl` 获取临时 URL 后通过 `resolveAttachmentItems` 转为 `blob: URL` 才能在页面展示
- 触发时机：`SettlementInfo.vue` 内部 `mounted` 判断 `waybillId` 和 `detail.transportType` 均就绪后触发一次；后续 `waybillId` 或 `transportType` 任一变化由对应 watch 重新触发，两者均不使用 `immediate`，避免初始化重复请求；图片存储在组件内部 data（`loadImages` / `unloadImages`），不通过 props 传入
- 图片数量：装货/卸货各支持多张（`el-carousel` 支持多 item 轮播）
- UI：`el-carousel` 原生轮播（多张图时可左右切换）；底部用 `el-radio-group` 做装货/卸货 Tab 切换
- 默认 Tab：有装货则默认装货；无装货有卸货则默认卸货；都无则展示空状态（icon + 空文案）
- 交互：只读，点击当前轮播图后调用 `el-image-x`（`ref.clickHandler()`）触发全屏预览，预览从点击的那张开始（`initialIndex` 与轮播索引对应）；支持下载按钮（权限同 `edit.vue` 的 `controlBtn.btnDownload`，由父组件通过 prop 传入）、上一张/下一张切换、底部标题文字
- 空状态文案：装货 Tab "暂无装货凭证照片"，卸货 Tab "暂无卸货凭证照片"
- 原表格装车净重/卸车净重/签收信息点击入口保留，不删除

锚点跳转使用 JS 方法 `scrollToAnchor(id)` 实现（原生 `<a href="#id">` 在嵌套滚动容器内无效，已废弃）：

```javascript
scrollToAnchor(id) {
  this.activeAnchor = id
  const el = document.getElementById(id)
  if (!el) return
  // 向上递归找到最近的 overflow:auto/scroll 容器
  const container = getScrollParent(el.parentElement)
  if (!container) {
    el.scrollIntoView({ block: 'start' })
  } else {
    // 累计 offsetTop，减去导航栏+页头高度偏移
    container.scrollTop = getOffsetToContainer(el, container) - 40 - 60 - 20
  }
}
```

导航按钮用 `<span>` + click 事件，不用 `<a>` 标签：

```html
<span
  v-for="item in anchorNavList"
  :key="item.id"
  :class="['anchor-nav-item', activeAnchor === item.id ? 'is-active' : '']"
  @click="scrollToAnchor(item.id)"
>{{ item.label }}</span>
```

高亮切换只跟随点击，`activeAnchor` 记录当前激活项，不监听滚动。
