# 【C149.5】易达宝 PC 端调整结算信息页面支持快速定位、装卸照片前置透出改动清单

## 需求概述

本次需求针对 PC 端「运单结算处理」中的两个页面：

1. 调整结算信息
2. 查看调整信息

核心优化：

- 在页面顶部增加一排固定导航按钮，点击跳转对应模块锚点。
- 将装卸货磅单图片前置展示在结算信息模块内，无需跳转二级页面。
- 本版本不涉及图片增删改，也不涉及保存行为变更。

> **与初版清单的差异说明（基于开发对齐）：**
> 1. 快速定位简化为"顶部固定按钮 + 点击锚点跳转"，不做双向联动滚动监听（原需求已删除）。
> 2. 结算信息模块抽成共用组件 `SettlementInfo.vue`，edit.vue 和 read.vue 共用；其余模块（运单信息、费用明细等）暂不抽组件。
> 3. 装卸货磅单图片直接加在 `SettlementInfo.vue` 组件内，组件通过 `readonly` prop 区分可编辑/只读态，通过 `showEditActions` prop 控制"修改重量"和"确认亏吨货损"按钮的显隐。
> 4. 不做结算信息双栏布局（左图右表单），磅单图片内嵌在结算信息区域展示即可。

## 二、功能点说明

### 2.1 功能点 1：顶部固定锚点导航

涉及模块：运单信息 / 结算信息 / 费用明细 / 风控预警 / 预审结果 / 操作记录

规则：

| 场景 | 规则 |
|---|---|
| 位置 | 页面顶部，固定在页面内容区最上方 |
| 初始化 | 默认高亮「运单信息」按钮 |
| 点击 | 点击按钮后对应按钮高亮，页面滚动到目标模块 |
| 吸顶 | 向下滚动时导航栏吸顶悬浮 |
| 滚动监听 | 不做，不随页面滚动自动切换高亮，避免卡顿 |
| 风控预警无数据 | 导航项是否隐藏待产品确认，建议隐藏或置灰 |

### 2.2 功能点 2：结算信息模块抽组件 + 磅单图片前置

**组件：`SettlementInfo.vue`**

接收 props：

| prop | 类型 | 说明 |
|---|---|---|
| `readonly` | Boolean | `true` = 只读（read.vue 使用），`false` = 可编辑（edit.vue 使用） |
| `showEditActions` | Boolean | 控制"修改重量"按钮和"确认亏吨货损"按钮的显示，edit.vue 传 `true`，read.vue 传 `false` |
| `formModel` | Object | 结算信息表单数据 |
| `detail` | Object | 运单/结算详情 |
| `dictionary` | Object | 数据字典 |
| `bindCardInfo` | Object | 绑卡信息 |
| `newCompanyOffLineDatas` | Array | 结算主体候选列表 |
| `splitMainInfo` | String | 委托转账主体 |
| `splitInfo` | String | 委托转账规则 |

emit 事件（edit 模式下需要）：

| 事件 | 说明 |
|---|---|
| `update:formModel` | 表单数据变更 |
| `open-edit-load-weight` | 点击"修改重量"按钮 |
| `open-confirm-loss` | 点击"确认亏吨货损"按钮 |
| `input-change` | 单价/结算数量/结算重量输入变更，用于联动运费计算 |

**磅单图片部分：**

- 在结算信息模块内新增装卸货磅单图片展示区域
- 装货/卸货 Tab 切换
- 页面初始化时预加载装货、卸货图片列表（调用 `getAttachmentsByWaybillId`）
- 有装货照片则默认展示装货；无装货有卸货则默认展示卸货；都无则展示空状态
- 图片只读，点击后调用现有 `el-image-x` 全屏预览
- 原表格中装车净重/卸车净重/签收信息的点击查看入口保留，不删除

## 三、现状分析与差异对比

### 3.1 edit.vue 与 read.vue 结算信息模块差异

经代码比对，两个文件的结算信息模块**存在以下实质差异**，不是纯粹的只读/可编辑区别：

| 位置 | edit.vue | read.vue |
|---|---|---|
| 卸货净重行 | 有"修改重量"按钮（带 `regulatoryReportingStatus`、`busStatus` 权限判断） | 无"修改重量"按钮 |
| 亏涨吨行 | 有"确认亏吨货损"操作按钮（带 `cargoCompensationConfirmFlag` 权限判断），取值来自 `formModel` | 无操作按钮，取值来自 `detail` |
| 备注区域 | 运单别名、备注可编辑 | 全部 disabled |
| 单价事件 | `@change` | `@input` |
| 毛重宽度 | 卸货毛重 width=95px | 卸货毛重 width=65px |

抽组件后，通过 `showEditActions` 控制按钮显隐，`readonly` 控制字段可编辑态，可以完整覆盖上述差异。

### 3.2 现有图片查看能力

当前两个页面已有：

- 运单信息表格中点击「装车净重」→ `handleLookImg(scope, '1')` → 查询附件 → `el-image-x` 预览
- 点击「卸车净重」→ `handleLookImg(scope, '2')`
- 点击「运单签收信息」→ `handleLookImg(scope, '3')`

本次磅单图片前置复用同一套 `getAttachmentsByWaybillId` 接口，改为页面初始化时预加载，原点击入口保留。

### 3.3 接口现状

| 接口 | 来源 | 当前用途 |
|---|---|---|
| `getAttachmentsByWaybillId` | `@/api/userBase.transport` | 点击时按 `operAttachmentType` 查询：1=装货、2=卸货、3=签收 |
| `postWaybillDetails` | `@/api/userBase.waybillSettlement` | 获取结算详情 |

本次需要在详情加载完成后额外调用两次（装货 + 卸货），或等待后端提供合并接口。

## 四、改动内容

### 4.1 新建 `SettlementInfo.vue`

从 edit.vue / read.vue 中抽取结算信息模块，包含：

- 标题行（结算信息）
- 第一行：单价、结算数量、结算重量、运费、结算方式
- 第二行：结算重量类型、装货净重+毛重、卸货净重+毛重（含修改重量按钮，受 `showEditActions` 控制）、下单重量
- 第三行：结算主体、收款开户行、收款银行卡号、亏涨吨（含确认亏吨货损按钮，受 `showEditActions` 控制）
- 委托转账行（splitFlag === '10' 时显示）
- 备注区域：运单别名、货源别名、备注
- **新增**：装卸货磅单图片展示区域

### 4.2 修改 `edit.vue`

1. 在页面内容区顶部插入锚点导航栏，按钮：运单信息 / 结算信息 / 费用明细 / 风控预警 / 预审结果 / 操作记录
2. 给各模块容器加 `ref` 锚点（见下表）
3. 将原结算信息模块替换为 `<SettlementInfo>` 组件，传入 `readonly=false`、`showEditActions=true` 及所需 props
4. 将原 `updBusCostInput`、`handleOpenEditLoadWeight`、`handleTodialog` 等方法通过 emit 事件与组件联动
5. 在 `getDetails` 成功后调用图片预加载方法，传给 `SettlementInfo` 组件
6. 原 `handleLookImg`、`handleWriteMessage` 等入口不删除

### 4.3 修改 `read.vue`

1. 同步插入锚点导航栏
2. 同步加 `ref` 锚点
3. 将原结算信息模块替换为 `<SettlementInfo>` 组件，传入 `readonly=true`、`showEditActions=false`
4. 在 `getDetails` 成功后调用图片预加载
5. 原 `handleLookImg`、`handleWriteMessage` 不删除

### 4.4 锚点 ref 规划

| 模块 | ref 名 |
|---|---|
| 运单信息 | `anchorWaybillInfo` |
| 结算信息 | `anchorSettlementInfo` |
| 费用明细 | `anchorCostDetail` |
| 风控预警 | `anchorRiskWarning` |
| 预审结果 | `anchorExamResult` |
| 操作记录 | `anchorOperationRecord` |

锚点跳转方法：

```js
scrollToAnchor(refName) {
  const el = this.$refs[refName]
  if (!el) return
  const target = el.$el || el
  target.scrollIntoView({ behavior: 'smooth', block: 'start' })
  this.activeAnchor = refName
}
```
