# 【C149.5】易达宝 PC 端调整结算信息页面支持快速定位、装卸照片前置透出改动清单

## 需求概述

本次需求针对 PC 端「运单结算处理」中的调整结算信息页面（edit.vue）：

- 在页面顶部增加一排固定导航按钮，点击跳转对应模块锚点
- 将装卸货磅单图片前置展示在结算信息模块内，采用左图右表单双栏布局，支持收起/展开
- 本版本不涉及图片增删改，也不涉及保存行为变更
- 结算信息模块抽成公共组件 `SettlementInfo.vue`，本期仅在 edit.vue 中使用；read.vue 完全不动，留待后续需求

## 二、功能点说明

### 2.1 功能点 1：顶部固定锚点导航（仅 edit.vue，read.vue 本期不做）

导航按钮列表：

| 按钮 | 显示规则 |
|---|---|
| 运单信息 | 始终显示 |
| 结算信息 | 始终显示 |
| 费用明细 | 始终显示 |
| 风控预警 | 动态：`highMidWaybillListPage.length > 0 && ['200','300','400'].includes(riskLevel)` 时显示，否则隐藏 |
| 预审结果 | 始终显示 |
| 操作记录 | 始终显示 |

交互规则：

| 场景 | 规则 |
|---|---|
| 位置 | 页面内容区顶部，`position: fixed`，不随页面滚动 |
| 初始化 | 默认高亮「运单信息」按钮 |
| 点击 | 对应按钮高亮，使用原生 `<a href="#id">` 跳转到目标模块 |
| 遮挡处理 | 页面内容区整体加 `padding-top` 等于导航栏高度 |
| 吸顶 | `position: fixed`，始终固定在顶部 |
| 滚动监听 | 不做（页面性能差，避免 scroll 事件），高亮只跟随点击 |

### 2.2 功能点 2：结算信息模块抽组件 + 磅单图片前置

**组件：`SettlementInfo.vue`**

接收 props：

| prop | 类型 | 说明 |
|---|---|---|
| `readonly` | Boolean | `true` = 只读（read.vue 使用），`false` = 可编辑（edit.vue 使用） |
| `showEditActions` | Boolean | 控制"修改重量"和"确认亏吨货损"按钮的显示，edit.vue 传 `true`，read.vue 传 `false` |
| `formModel` | Object | 结算信息表单数据 |
| `detail` | Object | 运单/结算详情 |
| `dictionary` | Object | 数据字典 |
| `bindCardInfo` | Object | 绑卡信息（edit 模式使用） |
| `newCompanyOffLineDatas` | Array | 结算主体候选列表 |
| `splitMainInfo` | String | 委托转账主体 |
| `splitInfo` | String | 委托转账规则 |
| `waybillId` | String | 运单号，组件内部 watch 到变化后自动调用磅单图片接口 |

emit 事件（edit 模式下需要）：

| 事件 | 说明 |
|---|---|
| `update:formModel` | 表单数据变更 |
| `open-edit-load-weight` | 点击"修改重量"按钮 |
| `open-confirm-loss` | 点击"确认亏吨货损"按钮 |
| `input-change` | 单价/结算数量/结算重量输入变更，用于联动运费计算 |

**结算信息模块整体布局：左图右表单**

结算信息模块采用左右双栏布局（`display: flex`）：

- 左侧：磅单图片区域，固定宽度
- 右侧：原有表单字段，`flex: 1` 占剩余宽度
- 右上角有"收起装卸照片"按钮，点击折叠左侧图片区域，表单自动回到全宽；再次点击展开

**磅单图片展开/收起规则：**

| 状态 | 说明 |
|---|---|
| 默认 | 始终展开，无论有无图片 |
| 无图片 | 左侧显示空状态文案，不自动收起 |
| 收起 | 用户手动点击右上角按钮，左侧隐藏，表单全宽；按钮文案变为"展开装卸照片" |
| 展开 | 用户手动点击再次展开；按钮文案变为"收起装卸照片" |

**磅单图片部分：**

- 新接口：后端提供，一次性返回装货+卸货图片，入参含运单号，路径联调时确认；由 `SettlementInfo.vue` 内部 watch `waybillId` 变化后自动调用，图片存储在组件内部 data（`loadImages` / `unloadImages`），不通过 props 传入
- 图片数量：装货最多一张，卸货最多一张
- UI：`el-carousel` 原生轮播；底部装货/卸货 Tab 切换
- 默认 Tab：有装货则默认装货；无装货有卸货则默认卸货；都无则展示空状态
- 交互：只读，点击使用 `el-image` 的 `preview-src-list` 放大查看
- 空状态文案：装货 Tab "暂无装货凭证照片"，卸货 Tab "暂无卸货凭证照片"
- 原表格装车净重/卸车净重/签收信息点击入口保留，不删除

## 四、改动内容

### 4.1 新建 `SettlementInfo.vue`

从 edit.vue 抽取结算信息模块，包含：

- 结算信息标题行 + 右上角"收起/展开装卸照片"按钮
- 主体：左右双栏布局（`display: flex`）
  - 左侧：磅单图片区域（固定宽度，默认展开）
  - 右侧：原有表单字段（`flex: 1`）
    - 第一行：单价、结算数量、结算重量、运费、结算方式
    - 第二行：结算重量类型、装货净重+毛重、卸货净重+毛重（含修改重量按钮，受 `showEditActions` 控制）、下单重量
    - 第三行：结算主体、收款开户行、收款银行卡号、亏涨吨（含确认亏吨货损按钮，受 `showEditActions` 控制）
    - 委托转账行（`splitFlag === '10'` 时显示）
    - 备注区域：运单别名、货源别名、备注

**磅单图片区域结构：**

```
[ el-carousel ]
  [ el-image（最多一张） ]
  空状态文案（无图片时显示）
-----------------------------
装货照片 Tab | 卸货照片 Tab
```

- Tab 切换只切换展示内容，不重新请求接口
- 点击图片：`el-image` 的 `preview-src-list` 放大
- 原模板代码注释保留，不直接删除

### 4.2 修改 `edit.vue`

1. 在 `examBox` 内容区顶部插入锚点导航栏
2. 导航按钮：运单信息 / 结算信息 / 费用明细 / **风控预警（动态显隐）** / 预审结果 / 操作记录
3. 给各模块容器加 `id` 锚点（见 4.4）
4. 将原结算信息模块（`<!-- 结算信息 -->` 至备注区域结束）注释保留，替换为 `<SettlementInfo>` 组件
5. `SettlementInfo` 传入：`readonly=false`、`showEditActions=true`、`waybillId` 及其余 props；磅单图片由组件内部自行加载，父组件只需传 `waybillId`
6. ~~在 `getDetails` 成功后调用磅单图片新接口，结果存入 `loadImages` / `unloadImages` data~~（已调整：图片加载逻辑封装在 `SettlementInfo.vue` 内部，`edit.vue` 无需处理）
7. 通过 emit 事件联动：`@input-change` → `updBusCostInput`、`@open-edit-load-weight` → `handleOpenEditLoadWeight`、`@open-confirm-loss` → `handleTodialog`
8. 原 `handleLookImg`、`handleWriteMessage` 等方法不删除

### 4.3 修改 `read.vue`

**本期完全不动，代码维持原样。**

`SettlementInfo.vue` 已按通用方式设计（含 `readonly` / `showEditActions` prop），待后续有需求改动查看页时，再按同样方式接入，届时对照清单 3.1 的差异逐项验证。

### 4.4 锚点规划（仅 edit.vue）

| 模块 | id |
|---|---|
| 运单信息 | `anchorWaybillInfo` |
| 结算信息 | `anchorSettlementInfo` |
| 费用明细 | `anchorCostDetail` |
| 风控预警 | `anchorRiskWarning` |
| 预审结果 | `anchorExamResult` |

锚点跳转使用原生 HTML 实现，无需 JS：

```html
<!-- 导航按钮 -->
<a href="#anchorSettlementInfo" @click="activeAnchor = 'anchorSettlementInfo'">结算信息</a>

<!-- 目标模块 -->
<div id="anchorSettlementInfo">...</div>
```

高亮切换只跟随点击，`activeAnchor` 记录当前激活项，不监听滚动。
