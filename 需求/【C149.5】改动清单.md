# 【C149.5】易达宝 PC 端调整结算信息页面支持快速定位、装卸照片前置透出改动清单

## 需求概述

本次需求针对 PC 端「运单结算处理」中的两个页面：

1. 调整结算信息
2. 查看调整信息

核心优化：

- 将原本需要从「运单详情 - 运单追踪」二级路径查看的装卸货照片，前置透出到调整/查看页面一级区域。
- 在调整/查看页面增加模块快速定位导航，支持点击定位到指定模块。
- 默认展开装卸货照片，支持一键收起，收起后页面恢复现有线上布局。
- 本版本不涉及图片增删改，也不涉及保存行为变更。

## 二、需求原文拆解

### 2.1 需求背景

业务在移动端预审流程中调整结算信息时，需要核对：

- 装货/卸货磅单照片
- 车货照等装卸货凭证
- 系统记录的装货重量、卸货重量、结算重量

当前问题：

- 装卸照片入口深，需要至少 3 步。
- 不利于用户边看图片边核对、修改结算数据。
- 页面模块较多，缺少快速定位。

### 2.2 功能点 1：调整结算一级页面支持查看装卸货照片

| 分类 | 需求 |
|---|---|
| 数据来源 | 自动关联读取原「运单详情 - 运单追踪」中的全部装卸货照片 |
| 照片范围 | 包含装货磅单、车货照等；卸货同理 |
| 初始化 | 页面加载时同步预加载该运单下所有装卸货图片资源 |
| 原入口 | 原「运单详情 - 运单追踪」查看入口保留 |
| 权限 | 前置图片只读，不支持增删改 |
| 布局 | 结算信息模块改为弹性双栏：左侧装卸图片，右侧结算信息字段 |
| 默认状态 | 照片默认展开 |
| 默认定位 | 默认定位装货照片；如果无装货照片但有卸货照片，则定位卸货照片 |
| 空状态 | 装卸货都无照片，左侧看板中央展示“暂无{装/卸}货凭证照片” |
| 图片组件 | 使用现有图片组件，全屏悬浮后支持放大、缩小、旋转 |
| 收起 | 点击“收起装卸货”隐藏左侧照片，页面布局恢复目前线上样式 |
| 保存 | 本版本不涉及保存操作行为 |

### 2.3 功能点 2：增加模块快速定位

涉及模块：

1. 运单信息
2. 结算信息
3. 费用明细
4. 风控预警
5. 预审结果
6. 操作记录

规则：

| 场景 | 规则 |
|---|---|
| 初始化 | 页面加载完成，默认激活「运单信息」 |
| 吸顶 | 页面向下滑动时，定位导航栏保持吸顶悬浮固定 |
| 点击定位 | 点击模块按钮后按钮点亮，页面滚动到目标模块 |
| 定位展示 | 目标模块优先展示；底部空间不足时，确保模块尽量完整透出 |
| 滚动监听 | 不做，需求已删除，避免页面卡顿 |

1. 运单信息
2. 结算信息
3. 备注信息

5. 付款方式

7. 预审结果

- 「卸车净重」点击调用 `handleLookImg(scope, '2')`

现有方法 `handleLookImg` 通过 `getAttachmentsByWaybillId` 查询附件，然后调用隐藏的 `el-image-x` 进行预览。

### 3.2 查看调整信息页现状

文件：`logisticsweb/src/views/userBase/waybillSettlement/read.vue`

结构与 `edit.vue` 高度相似，也已有：

- 运单信息
- 结算信息

### 3.3 图片能力现状

当前项目已有 `el-image-x` 组件，能力包括：

- 基于 `el-image` 二次开发。
- 支持 `preview-src-list`。
- 支持上一张、下一张事件。
- 支持插槽展示标题。
- 当前结算页已用它实现全屏悬浮预览。

因此本期“使用现有图片组件”的可行方案是：

- 左侧看板展示缩略图/列表/主图。
- 点击图片后仍调用现有 `el-image-x` 打开全屏预览。
- 放大、缩小、旋转等能力仍依赖现有预览层。

### 3.4 接口现状

当前结算页已有接口调用：

| 接口方法 | 来源 | 用途 |
|---|---|---|
| `postWaybillDetails` | `@/api/userBase.waybillSettlement` | 获取结算详情 |
| `getAttachmentsByWaybillId` | `@/api/userBase.transport` | 当前按装货/卸货/签收查询附件 |
| `dcsOperationRecordPagingexport` | `@/api/userBase.waybillSettlement` | 获取操作记录 |

当前 `handleLookImg` 是点击时按类型查询：

- `operAttachmentType: '1'` 装货
- `operAttachmentType: '2'` 卸货
- `operAttachmentType: '3'` 签收

需求要求“页面初始化加载时同步预加载该运单下所有装卸货图片资源”，这可能有两种实现：

1. 前端初始化时分别调两次 `getAttachmentsByWaybillId`：装货一次、卸货一次。
2. 后端提供新接口一次返回全部装卸货照片。

需求文档中提到“辛苦姚奇老师提供接口支持”，因此接口是否复用现有 `getAttachmentsByWaybillId` 不能默认确认，需要后端明确。

## 四、业务规则

### 4.1 装卸货照片加载规则

| 场景 | 规则 |
|---|---|
| 页面初始化 | 加载运单详情后，根据 `waybillId`、`transportType` 获取装卸货照片 |
| 有装货照片 | 左侧默认展示装货照片 |
| 无装货照片，有卸货照片 | 左侧默认展示卸货照片 |
| 装卸货都无照片 | 展示空状态文案 |
| 接口失败 | 建议左侧展示加载失败/暂无照片，不阻断页面主流程 |
| 照片只读 | 不显示上传、删除、编辑入口 |
| 原查看入口 | 运单信息中的重量点击查看、查看跟踪信息入口保留 |

### 4.2 照片分类规则

建议前端内部统一维护两个集合：

| 字段建议 | 含义 |
|---|---|
| `loadPhotoList` | 装货照片列表 |
| `unloadPhotoList` | 卸货照片列表 |
| `activePhotoType` | 当前选中的照片类型：装货/卸货 |
| `activePhotoIndex` | 当前展示照片索引 |
| `settlementPhotoExpanded` | 左侧照片是否展开 |

注意：字段名仅为前端建议，最终以后端接口返回结构为准。

### 4.3 空状态规则

需求原文是：

> 装卸货都无照片，左侧看板中央展示提示：“暂无{装/卸}货凭证照片”

这里有歧义：

- 如果装货无、卸货有：默认定位卸货，不展示装货空状态。
- 如果装货有、卸货无：默认定位装货。
- 如果用户手动切换到无照片的类型，是否展示“暂无装货凭证照片 / 暂无卸货凭证照片”需要确认。
- 如果装卸货都无，默认文案中的 `{装/卸}` 应展示“装卸货”还是根据默认 Tab 展示“装货”，需确认。

当前建议：

| 场景 | 建议文案 |
|---|---|
| 当前选中装货且无照片 | 暂无装货凭证照片 |
| 当前选中卸货且无照片 | 暂无卸货凭证照片 |
| 装卸货都无照片且未选中类型 | 暂无装卸货凭证照片 |

### 4.4 双栏布局规则

| 状态 | 布局 |
|---|---|
| 默认展开 | 左侧装卸货照片看板 + 右侧结算信息字段 |
| 点击收起 | 隐藏左侧照片看板，结算信息恢复当前线上单栏/整宽布局 |
| 再次展开 | 恢复左侧照片看板和右侧结算信息布局 |

建议只影响「结算信息」模块，不影响「运单信息」「费用明细」等其他模块。

### 4.5 快速定位规则

| 模块 | 对应现有页面区域 |
|---|---|
| 运单信息 | `.waybill-message` |
| 结算信息 | `.settlement-message` |
| 费用明细 | `CostMessageDetails` 前的标题区域 |
| 风控预警 | `RiskWarning` 区域 |
| 预审结果 | `ExamResult` 区域 |
| 操作记录 | `EditMessageDetails` 区域 |

点击导航：

- 设置当前激活模块。
- 使用 `scrollIntoView` 或容器内滚动计算定位。
- 因为页面外层存在 `.examBox` / `.content` 等容器，需确认实际滚动容器是 `window` 还是页面内部容器。
- 不做滚动监听自动切换，避免卡顿。

### 5.2 PC 端查看调整信息页：`read.vue`

查看页应同步实现：

- 快速定位导航。
- 结算信息双栏布局。
- 左侧装卸货照片只读展示。
- 默认展开，支持收起。
- 原运单追踪入口保留。
- 原重量点击查看图片入口保留。

区别：

- 查看页不涉及任何编辑能力。
- 收起/展开只影响页面展示，不影响数据。
- 查看页本身不涉及保存/提交，不应引入编辑页保存相关逻辑。

### 5.3 公共化建议

由于 `edit.vue` 和 `read.vue` 结构重复度高，本需求如果直接分别写逻辑，容易产生重复代码。

建议新增或抽取内部组件，但是否创建新文件需结合开发成本：

#### 方案 A：在两个页面内分别实现

优点：

- 改动直观。
- 不影响其他页面。
- 适合快速交付。

缺点：

- 重复代码较多。
- 后续维护成本较高。

#### 方案 B：抽取组件

建议组件：

| 组件 | 作用 |
|---|---|
| `SettlementAnchorNav.vue` | 快速定位导航 |
| `SettlementLoadUnloadPhotos.vue` | 装卸货照片看板 |

优点：

- `edit.vue` / `read.vue` 复用。
- 逻辑集中，后续维护更好。

缺点：

- 新增文件，需要额外验证组件引用。
- 如果本次工期紧，初始成本略高。

推荐：方案 B。

原因是本需求明确同时影响调整页和查看页，且两个功能（导航、照片看板）都是纯 UI/交互能力，适合组件化，避免两处复制。

但根据项目规则“非必要不创建文件”，如果希望最小改动，也可以采用方案 A。

## 十、风险点

| 风险 | 说明 | 建议 |
|---|---|---|
| 初始化加载图片影响性能 | 图片数量多时首屏加载变慢 | 接口只预取列表，图片可懒加载；看板缩略图限制尺寸 |
| 接口范围不明确 | “全部装卸货照片”可能和现有点击接口返回不一致 | 先确认后端接口范围 |
| 双栏导致表单过窄 | 原 `el-col :span="6"` 在右侧容器中可能拥挤 | 右侧布局可能需要调整列宽，或左侧设置可收起 |
| 快速定位容器不明确 | 页面可能不是 `window` 滚动 | 开发时确认 `.examBox` / 页面主体滚动容器 |
| 风控预警条件渲染 | 无风控时模块不存在 | 产品确认导航项隐藏还是保留 |
| 两个页面重复改造 | `edit.vue` 和 `read.vue` 结构类似，容易漏改 | 建议抽组件复用 |
| 旧逻辑误删 | 原重量点击看图、追踪入口需保留 | 仅新增前置入口，不删除旧入口 |
| `el-image-x` 全屏层级 | 吸顶导航可能遮挡预览层 | 确认 z-index |
| 图片分类标题 | 接口 `label` 是否满足 UI 展示 | 后端确认返回 label |

## 十一、开发顺序建议

1. 确认接口
   - 是否新增接口。
   - 返回结构。
   - 装卸货照片范围。
   - 不同运输类型状态节点规则。
2. 确认 UI 细节
   - 左侧照片区域尺寸。
   - 缩略图展示方式。
   - 收起按钮文案。
   - 空状态文案。
   - 无风控时导航项行为。
3. 实现公共交互
   - 快速定位导航。
   - 锚点定位方法。
   - 吸顶样式。
   - 当前点击高亮。
4. 实现照片数据加载
   - 初始化加载装货/卸货照片。
   - 数据归一化。
   - 默认选中规则。
   - 异常/空状态处理。
5. 改造调整页 `edit.vue`
   - 增加导航。
   - 增加锚点。
   - 结算信息双栏布局。
   - 接入照片看板。
   - 保留原入口。
6. 改造查看页 `read.vue`
   - 同步改造。
   - 确保只读。
7. 验证与回归
   - 功能验证。
   - 边界验证。
   - 保存/提交回归。
   - ESLint 验证。

## 十二、建议实施方案

推荐采用组件化方案：

| 组件 | 用途 |
|---|---|
| `SettlementAnchorNav` | 快速定位导航 |
| `LoadUnloadPhotoPanel` | 装卸货照片看板 |

理由：

- 调整页和查看页都需要。
- 逻辑一致，降低漏改风险。
- 后续如果其他结算页也要复用，扩展方便。

如果本次希望最小改动、避免新增文件，可以先在 `edit.vue` / `read.vue` 内实现，但需要接受重复代码。
